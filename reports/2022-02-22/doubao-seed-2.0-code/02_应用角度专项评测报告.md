# MindX 项目 - 应用角度专项评测报告

## 1. 应用场景概览

MindX 作为一个具备思考能力的 AI 个人助理，其应用场景覆盖个人生活、工作效率、内容创作等多个维度。

### 1.1 核心应用场景矩阵

| 场景大类 | 具体场景 | 核心能力依赖 |
|---------|---------|-------------|
| **个人助理** | 日程管理、备忘录、天气查询、消息提醒 | 本地模型 + 技能系统 + 定时任务 |
| **内容创作** | 文章写作、文案创作、翻译、内容改写 | 能力系统 + 云端大模型 + 文件技能 |
| **编程辅助** | 代码编写、调试、优化、代码审查 | 编程能力 + 文件技能 |
| **多平台沟通** | 跨平台消息转发、统一回复、消息管理 | 多渠道接入 Gateway |
| **知识管理** | 信息整理、笔记记录、知识检索 | 记忆系统 + 向量搜索 |
| **生活服务** | 日历、邮件、联系人、文件操作 | 内置技能生态 |

#### 2.3.2 记忆点数据结构

```go
type MemoryPoint struct {
    ID             int       // 记忆点唯一标识
    Keywords       []string  // 关键字列表
    Content        string    // 原始内容
    Summary        string    // 自动生成的摘要
    Vector         []float64 // 向量表示（用于相似度搜索）
    ClusterID      int       // 聚类 ID（用于记忆分组）
    TimeWeight     float64   // 时间权重（越新越高）
    RepeatWeight   float64   // 重复权重（提及次数越多越高）
    EmphasisWeight float64   // 强调权重（用户强调的内容）
    TotalWeight    float64   // 综合权重
    CreatedAt      time.Time // 创建时间
    UpdatedAt      time.Time // 更新时间
}
```

### 2.5 能力配置系统

#### 2.5.1 预定义能力

MindX 支持通过 YAML 配置文件定义能力，预设能力包括：

| 能力名称 | 绑定模型 | 关联技能 | 功能描述 |
|---------|---------|---------|---------|
| **writing** | qwen3.5-plus | write_file, read_file | 网络爆文、文章写作、文案创作 |
| **coding** | GLM-4-0520 | write_file, read_file | 代码编写、调试、优化 |
| **translate** | qwen3:1.7b | - | 语言翻译、文本翻译 |

#### 2.5.2 能力配置特性

```yaml
capabilities:
  - name: writing
    title: 网络爆文
    icon: EditIcon
    description: "长文写作能力，撰写文章、写作内容、改写文章、内容创作、文案写作"
    model: "qwen3.5-plus"
    system_prompt: "# 网络爆款文案智能体 核心设定\n你是一个专注于创作「网络爆款文案」的资深智能体。"
    tools: ["write_file", "read_file"]
    enabled: true
```

**能力调用方式**：
1. **自动匹配**：根据用户意图自动匹配合适的能力
2. **前缀调用**：使用 `/能力名 问题` 格式显式调用，如 `/writing 写一篇文章`

## 4. 实际应用场景深度分析

### 4.1 场景一：个人日常助理

**典型使用流程**：
1. 用户："明天天气怎么样？"
2. 左脑（本地模型）快速理解意图
3. 匹配天气查询技能
4. 调用技能获取天气信息
5. 返回自然语言回答

**评测要点**：
- ✅ 响应速度快（本地模型）
- ✅ 零 Token 消耗
- ✅ 隐私安全（数据本地处理）

**适用人群**：
- 关注隐私的个人用户
- 日常需要处理简单任务的用户
- 希望降低 AI 使用成本的用户

### 4.2 场景二：内容创作

**典型使用流程**：
1. 用户："/writing 写一篇关于 AI 的文章"
2. 识别能力前缀，激活 writing 能力
3. 绑定 qwen3.5-plus 模型
4. 注入写作专属 System Prompt
5. 可结合 write_file 技能保存结果

**评测要点**：
- ✅ 能力配置灵活
- ✅ 可绑定最优模型
- ✅ 技能组合增强能力
- ⚠️ 依赖云端大模型

**适用人群**：
- 内容创作者

**典型使用流程**：

**评测要点**：
- ✅ 多渠道统一管理
